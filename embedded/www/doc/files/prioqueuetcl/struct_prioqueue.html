Description

Welcome to Struct, a set of packages providing various data structures to Tcl, and additional operations for existing Tcl structures.

This manpage documents the C implementation of the struct::prioqueue package. It provides the same API as the Tcl implementation, although details like the exact wording of error messages may differ.

Class API

The ::struct namespace contains a single command with multiple methods for the creation of priority queue objects. These are:

::struct prioqueue new ?options...?

This class method creates a new priority queue instance with an associated global Tcl command and returns the fully qualified name of this command. The name is auto-generated.

::struct prioqueue create name ?options...?

This class method creates a new priority queue instance with an associated global Tcl command and returns the fully qualified name of this command. The name is chosen by the user and provided as the argument of the method.

In both cases options can be used to specify the key type and ordering of the instance. The key type options are:

-integer

The key is an integer number. This is the default type if it was not specified at instance creation. The default ordering for this type is -decreasing.

-real

The key is a floating point number. The default ordering for this type is -decreasing.

-ascii

The key is a plain string. The default ordering for this type is -increasing.

-dict
-dictionary

The key is a Tcl dictionary. The default ordering for this type is -increasing.

The key ordering options are:

-increasing

The data in the queue is sorted in ascending order.

-decreasing

The data in the queue is sorted in descending order.

The default ordering is type specific, see above.

Instance API

All instances of queues created through the Class API support the following API:

prioqueueObj clear

This method removes all elements from the priority queue.

The method returns the empty string as its result.

prioqueueObj destroy

This method destroys the priority queue instance and the associated Tcl command, releasing any associated resurces.

The method returns the empty string as its result.

prioqueueObj peek ?count?

This method returns the first count items found in the priority queue, without removing them from the queue. The associated priorites are not returned. The count defaults to 1.

If more than one element is requested the result will be a list of elements, with the front element of the queue as the first element of the result. If only one element is requested the result is that element.

An error will be thrown if the queue does not hold enough elements to satisfy the request, i.e. less than count elements.

prioqueueObj peekpriority ?count?

This method behaves like method peek, except that it returns the priorities instead of the elements themselves.

prioqueueObj pop ?count?

This method behaves like method peek, except that it removes the returned element(s) as well.

prioqueueObj put item prio ?item prio ...?

This method adds the item or items specified to the priority queue. The prio value associated with each item must be a valid priority key for this type of queue (See the Class API). An error will be thrown if this is not the case, and nothing will be added, even if other key/item combinations are valid.

Items are inserted at their priority ranking. Items with equal priority are added in the order they were added.

prioqueueObj remove element

This method removes the selected element from the priority queue.

The method returns the empty string as its result.

prioqueueObj size

This method returns the number of elements stored in the priority queue.

Changes

Changes for version 2

Version 2 introduces incompatible changes to the API of priority queue objects, therefore the change to the major version number.

The changes in detail are:

  1. Version 2 requires Tcl 8.5 or higher. Tcl 8.4 or less is not supported anymore.

  2. Instance creation syntax has changed. For comparison, the old syntax is

    	struct::prioqueue FOO       ; # or
	set foo [struct::prioqueue]

    whereas the new is

    	struct prioqueue create FOO ; # or
	set foo [struct prioqueue new]

    Similarly, the options specifying key type and order are now found after the instance name, not before it.

  3. The method get has been renamed to pop.
